# PcMqttAgent 🖥️📡

Легковесное WPF-приложение для Windows 10/11, которое работает в системном трее, публикует состояние ПК в MQTT и принимает базовые команды удалённого управления.

Проект удобно подключать к Home Assistant, Node-RED, MajorDoMo и другим системам умного дома или удалённого мониторинга.

## ✨ Возможности

- 🖥️ **Работа в фоне** — скрытое основное окно и иконка в системном трее с контекстным меню.
- 📊 **Мониторинг ПК** — загрузка CPU/RAM, uptime и температуры через `LibreHardwareMonitor`.
- 📡 **MQTT-интеграция**:
  - автоматическое переподключение при обрыве связи;
  - поддержка **Last Will and Testament (LWT)** для публикации `OFFLINE` при аварийном отключении;
  - публикация показателей с настраиваемым интервалом;
  - retained-сообщения для актуального состояния после переподключения клиента.
- 🎮 **Удалённое управление** — выключение и перезагрузка ПК через MQTT-топик команд.
- ⚙️ **Настройка через JSON** — параметры MQTT, интервала публикации и логирования хранятся в `appsettings.json`.
- 📝 **Логирование** — файловые логи с ежедневной ротацией через `Serilog`.

## 📋 Требования

- **ОС:** Windows 10 или Windows 11.
- **.NET:** .NET 9.0 Runtime для запуска или .NET 9.0 SDK для сборки.
- **Права:** для чтения некоторых датчиков и выполнения команд выключения/перезагрузки рекомендуется запуск от имени администратора.
- **MQTT-брокер:** например Mosquitto, EMQX, Home Assistant MQTT broker или другой совместимый брокер.

## 🚀 Установка и запуск

### 1. Клонирование репозитория

```bash
git clone https://github.com/ВАШ_НИК/WinMQTT-client.git
cd WinMQTT-client/PcMqttAgent
```

### 2. Настройка MQTT

Откройте `PcMqttAgent/appsettings.json` и укажите параметры вашего MQTT-брокера:

```json
{
  "Mqtt": {
    "Server": "mqtt-server.lan",
    "Port": 1883,
    "ClientId": "MAINPC",
    "User": "your_username",
    "Password": "your_password",
    "BaseTopic": "MAINPC"
  },
  "Publisher": {
    "IntervalSeconds": 60
  },
  "Logging": {
    "FilePath": "logs/agent-.log",
    "LogLevel": "Information"
  }
}
```

> ⚠️ Не коммитьте реальные логины и пароли от MQTT-брокера в публичный репозиторий.

### 3. Сборка и запуск

Убедитесь, что терминал или Visual Studio Code запущены от имени администратора, затем выполните:

```bash
dotnet restore
dotnet build
dotnet run
```

Для запуска из корня репозитория можно указать проект явно:

```bash
dotnet run --project PcMqttAgent/PcMqttAgent.csproj
```

## ⚙️ Параметры конфигурации

| Параметр | Описание | Пример |
| --- | --- | --- |
| `Mqtt.Server` | Адрес MQTT-брокера | `mqtt-server.lan` |
| `Mqtt.Port` | Порт MQTT-брокера | `1883` |
| `Mqtt.ClientId` | MQTT Client ID приложения | `MAINPC` |
| `Mqtt.User` | Имя пользователя MQTT | `your_username` |
| `Mqtt.Password` | Пароль MQTT | `your_password` |
| `Mqtt.BaseTopic` | Базовый топик для публикаций и команд | `MAINPC` |
| `Publisher.IntervalSeconds` | Интервал публикации метрик в секундах | `60` |
| `Logging.FilePath` | Путь к файлам логов | `logs/agent-.log` |
| `Logging.LogLevel` | Минимальный уровень логирования | `Information` |

## 📡 Структура MQTT-топиков

Все топики строятся от значения `Mqtt.BaseTopic`. В примерах ниже используется базовый топик `MAINPC`.

### 📤 Публикуемые данные: Agent → Broker

| Топик | Описание | Пример значения | Retain |
| --- | --- | --- | --- |
| `MAINPC/status` | Статус подключения | `ONLINE` / `OFFLINE` | ✅ Да |
| `MAINPC/info/version` | Версия приложения | `1.0.0.0` | ✅ Да |
| `MAINPC/info/uptime` | Время работы системы | `00.05:30:15` | ✅ Да |
| `MAINPC/info/cpu` | Загрузка процессора | `15.5%` | ✅ Да |
| `MAINPC/info/ram` | Загрузка оперативной памяти | `42.1%` | ✅ Да |
| `MAINPC/info/temp/cpu` | Температура CPU | `45.0°C` | ✅ Да |
| `MAINPC/info/temp/gpu` | Температура GPU | `38.0°C` | ✅ Да |

Если датчик недоступен, соответствующая метрика может не публиковаться до появления значения.

### 📥 Команды: Broker → Agent

Отправьте payload в топик `MAINPC/command`:

| Payload | Действие |
| --- | --- |
| `shutdown` | Немедленное выключение ПК |
| `reboot` | Немедленная перезагрузка ПК |
| `restart` | Немедленная перезагрузка ПК |

Пример через `mosquitto_pub`:

```bash
mosquitto_pub -h mqtt-server.lan -t MAINPC/command -m reboot
```

## 🔄 Автозапуск с правами администратора без UAC

Чтобы приложение запускалось автоматически при входе в систему с повышенными правами и без всплывающего окна UAC, используйте Планировщик заданий Windows:

1. Откройте **Планировщик заданий** (`taskschd.msc`).
2. Нажмите **Создать задачу...** (не **Создать простую задачу...**).
3. На вкладке **Общие**:
   - задайте имя, например `PcMqttAgent`;
   - включите **Выполнять с наивысшими правами**;
   - выберите запуск только при входе пользователя, если приложение должно работать в пользовательской сессии и показывать иконку в трее.
4. На вкладке **Триггеры** добавьте триггер **При входе в систему**.
5. На вкладке **Действия** добавьте действие **Запустить программу** и укажите полный путь к `PcMqttAgent.exe`.
6. Нажмите **ОК** и проверьте запуск задачи вручную.

## ⚠️ Известные ограничения оборудования

- **Температура CPU на некоторых мобильных AMD Ryzen.** Из-за ограничений драйверов и BIOS датчик `Core (Tctl/Tdie)` может возвращать некорректные значения или быть недоступен. Это ограничение оборудования/драйверов, а не приложения.
- **Температура встроенной графики (iGPU).** Встроенные видеокарты AMD Radeon Graphics и Intel UHD часто не имеют отдельного температурного датчика, поэтому метрика GPU может не публиковаться.
- **Права доступа к датчикам.** Часть сенсоров может быть видна только при запуске приложения от имени администратора.
